## Annexe S — Documentation technique : Fuzzy search des ingrédients

**Référencée en :** section 2.5.2

**Problème résolu :** éviter la création de doublons d'ingrédients
("citon" alors que "citron" existe) tout en permettant des noms composés
("fraise des bois" alors que "fraise" existe).

**Algorithme à deux requêtes :**

```
Utilisateur tape "citon"
  │
  ├── Requête 1 : WHERE nom CONTAINS "citon"
  │   → [] (vide — normal pour une faute de frappe)
  │
  ├── Moins de 3 résultats → Requête 2 de rattrapage
  │   WHERE nom CONTAINS "cit"  (3 premières lettres)
  │   → ["citron", "citronnade", "citronnelle"]
  │
  └── Tri par distance de Levenshtein
        "citon" → "citron"      : distance 1  ✅ suggéré en #1
        "citon" → "citronnade"  : distance 5  — en #2
        "citon" → "citronnelle" : distance 6  — en #3
```

**Distance de Levenshtein :**
Nombre minimal d'opérations (insertion, suppression, substitution)
pour transformer un mot en un autre.
"citon" → "citron" = 1 insertion du "r" = distance 1.

**Normalisation au singulier :**
Tout ingrédient est normalisé avant sauvegarde :
le `s` final est supprimé sauf pour les exceptions
(`riz`, `noix`, `pois`, `fois`, `bois`, `fois`, `fois`...).
"Tomates" → "tomate", "citrons" → "citron", "riz" → "riz" (inchangé).

**Règle métier :**
Si l'ingrédient saisi correspond exactement à un existant (distance 0),

## Annexe U — Documentation technique : Pipeline Sharp/WebP

**Référencée en :** section 2.5.2

**Problème résolu :** réduire le poids des images uploadées (−70% en
moyenne) sans écrire le fichier original sur le disque.

**Flux de traitement :**

```
Utilisateur upload une image (JPEG/PNG/WebP)
  │
  ├── Multer (storage: memoryStorage())
  │   L'image est conservée en mémoire RAM (Buffer)
  │   Jamais écrite sur le disque en format original
  │
  ├── Sharp (traitement en mémoire)
  │   .resize(800, 600, { fit: 'cover' })   ← recadrage centré
  │   .webp({ quality: 80 })                ← conversion WebP
  │   .toBuffer()                            ← retour Buffer
  │
  └── Écriture finale
      fs.writeFile(`/uploads/${uuid}.webp`)
      → imageUrl stockée en BDD : "/uploads/uuid.webp"
```

**Gains mesurés :**

| Format original | Taille avant | Taille après WebP | Réduction |
|----------------|-------------|-------------------|-----------|
| JPEG 2000×1500 | ~800 Ko | ~240 Ko | −70% |
| PNG 1200×900 | ~1,2 Mo | ~180 Ko | −85% |
| WebP 800×600 | ~150 Ko | ~120 Ko | −20% |

**Formats acceptés en entrée :** JPEG, JPG, PNG, WebP
(configuré dans Multer via `fileFilter` avec vérification du MIME type).

## Annexe AA — Journal des difficultés rencontrées

**Référencée en :** section T.4.4

| # | Difficulté | Sprint | Description | Solution |
|---|-----------|--------|-------------|----------|
| 1 | Fork GitHub read-only | S1 | L'organisation O-clock-Francfort est en lecture seule — push impossible depuis le repo principal | Remote `perso` (Nathalie-Leduc/cines-delices) + sync fetch + merge + push |
| 2 | Prisma migrations P3005 | S1 | BDD Railway désynchronisée — migrations échouaient avec code P3005 | Reset DB prod + replay intégral `prisma migrate deploy` |
| 3 | VITE_API_URL non injectée | S1 | Variable Vite non définie avant le build — toutes les requêtes API pointaient vers localhost en prod | Ajout variable Railway + rebuild complet (commit vide) + vérification bundle JS |
| 4 | Fuzzy search trop agressif | S2 | "fraise des bois" impossible à créer car "fraise" existait déjà — normalisation bloquante | Double requête (contains exact + préfixe 3 lettres) + tri Levenshtein + seuil de distance |
| 5 | parseTimeToMinutes | S2 | Formats libres (1h10, 1:10, 30min) stockés comme 0 en BDD | Regex multi-patterns couvrant tous les formats + 8 tests unitaires |
| 6 | Images non affichées prod | S2 | imageURL relative en dev, API_BASE_URL absent en prod | Variable `API_BASE_URL` + préfixage systématique dans les controllers |
| 7 | Merge FK RESTRICT | S2 | Impossible de supprimer un ingrédient utilisé dans des recettes (contrainte FK) | Transaction Prisma atomique : `updateMany` (repoint) puis `delete` (source) |
| 8 | Conflits Git CreateRecipe.jsx | S2–S3 | Fichier 1800+ lignes — conflits récurrents lors des merges | Refactoring en composants thématiques + extraction `recipeUtils.js` |
| 9 | logout() non centralisé | S2 | Appel direct `localStorage.removeItem()` dans les composants — state React non mis à jour | Centralisation `logout()` dans `AuthContext` avec `useCallback` |
| 10 | TMDB doublons media | S2 | Un film et une série pouvaient avoir le même `tmdb_id` — risque de doublon | Contrainte composite `@@unique([tmdbId, type])` dans le schéma Prisma |
